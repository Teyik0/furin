# File-Based Routing

Furin automatically generates routes from your `src/pages/` directory. Each file becomes a route.

## Basic Routes

```
src/pages/index.tsx        → /
src/pages/about.tsx        → /about
src/pages/blog/index.tsx   → /blog
```

## Dynamic Routes

Wrap a segment in brackets for dynamic parameters:

```
src/pages/blog/[slug].tsx  → /blog/:slug
src/pages/[...all].tsx     → /* (catch-all)
```

Access params in your loader:

```tsx
export default route.page({
  loader: async ({ params }) => {
    const post = await db.getPost(params.slug)
    return { post }
  },
  component: ({ post }) => <article>{post.title}</article>,
})
```

## Layouts

Files named `_route.tsx` define layouts for their directory:

```
src/pages/blog/_route.tsx   ← layout for all /blog/* routes
src/pages/blog/index.tsx    ← uses the blog layout
src/pages/blog/[slug].tsx   ← uses the blog layout
```

### Nesting Layouts

Layouts can be nested by specifying a `parent`:

```tsx
// src/pages/blog/_route.tsx
import { route as rootRoute } from "../root"

export const route = createRoute({
  parent: rootRoute,
  layout: ({ children }) => (
    <div className="flex">
      <Sidebar />
      <main>{children}</main>
    </div>
  ),
})
```

## Type-Safe Navigation

Furin generates a `RouteManifest` type so navigation is fully typed:

```tsx
import { Link } from "@teyik0/furin/link"

// TypeScript knows /blog accepts a `page` search param
<Link to="/blog" search={{ page: 2 }} />
```

Add `furin-env.d.ts` to your `tsconfig.json` includes:

```json
{
  "include": ["src/**/*", "furin-env.d.ts"]
}
```
