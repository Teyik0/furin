# Plugins

Furin's build system is Bun's bundler under the hood. You can pass any Bun plugin to process assets at build time and at runtime.

## Adding Plugins

Declare plugins in `furin.config.ts`:

```ts
// furin.config.ts
import { defineConfig } from "@teyik0/furin/config"
import tailwind from "bun-plugin-tailwind"

export default defineConfig({
  pagesDir: "./src/pages",
  plugins: [tailwind],
})
```

Plugins run before Furin's internal client transform, so they can process any file type your pages import.

## Dev Server Plugins

For development (hot reload), plugins also need to be declared in `bunfig.toml`:

```toml
# bunfig.toml
[serve.static]
plugins = ["bun-plugin-tailwind"]
env = "FURIN_PUBLIC_*"
```

And registered at runtime via `Bun.plugin()` in your server entry for SSR:

```ts
// src/server.ts
import { furin } from "@teyik0/furin"
import { Elysia } from "elysia"
import myPlugin from "./lib/my-plugin.ts"

Bun.plugin(myPlugin)

const app = new Elysia()
  .use(await furin({ pagesDir: "./src/pages" }))
  .listen(3000)
```

## Writing a Plugin

A Furin/Bun plugin is a standard `Bun.BunPlugin`:

```ts
// src/lib/my-plugin.ts
const myPlugin: Bun.BunPlugin = {
  name: "my-plugin",
  setup(build) {
    build.onLoad({ filter: /\.txt$/ }, async (args) => {
      const text = await Bun.file(args.path).text()
      return {
        contents: `export default ${JSON.stringify(text)}`,
        loader: "js",
      }
    })
  },
}

export default myPlugin
```

## MDX Support

MDX is not included by default but is easy to add:

```ts
// src/lib/bun-mdx-plugin.ts
import { compile } from "@mdx-js/mdx"
import remarkGfm from "remark-gfm"

const MDX_FILTER = /\.mdx$/

const mdxPlugin: Bun.BunPlugin = {
  name: "furin-mdx",
  setup(build) {
    build.onLoad({ filter: MDX_FILTER }, async (args) => {
      const source = await Bun.file(args.path).text()
      const compiled = await compile(source, {
        outputFormat: "program",
        remarkPlugins: [remarkGfm],
      })
      return { contents: String(compiled), loader: "js" }
    })
  },
}

export default mdxPlugin
```

Add it to both `furin.config.ts` and `bunfig.toml`, and call `Bun.plugin(mdxPlugin)` in `server.ts`.

## Available Plugins

| Plugin | Purpose |
|---|---|
| `bun-plugin-tailwind` | Tailwind CSS v4 |
| `@mdx-js/mdx` (custom) | MDX files |
| `bun-plugin-yaml` | Import YAML files |
